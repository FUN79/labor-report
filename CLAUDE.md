## 7. 勞報單自動化系統

### 為什麼要做這個？
FUN星球每次辦活動後，需要幫外部表演夥伴（如瑋德、阿海、BJ、Gary）開立勞務報酬單。原本流程繁瑣：夥伴要填紙本或把資料零散傳給 Lulu，Lulu 再手動整理、製作 PDF、來回確認，耗費大量溝通成本。

目標：讓夥伴自己線上填資料、簽名、上傳身分證；Lulu 只需在試算表填金額後一鍵產出 PDF，省掉所有來回。

### 系統架構
- **前端表單**：GitHub Pages 靜態網頁（`https://fun79.github.io/labor-report/`）
  - 程式碼位置：`/Users/apple/Downloads/Linda's Agent-嘻嘻/勞報單系統/index.html`
  - GitHub repo：`github.com/FUN79/labor-report`
- **後端（Apps Script）**：接收表單資料、存圖片到 Drive、寫入 Sheets、產出 PDF
  - 程式碼位置：`/Users/apple/Downloads/Linda's Agent-嘻嘻/勞報單系統/Code.gs`
  - 執行身分：`funstoryplanet@gmail.com`

### Sheets 欄位順序
A 送出時間 / B 姓名 / C 電話 / D 國籍 / E 地址 / F 所得類別 / G 身分證字號 / H 電子簽名 / I 身分證正面 / J 身分證反面 / K 勞務內容（Lulu 填）/ L 應付總額（Lulu 填）/ M 代扣所得稅（公式）/ N 二代健保（公式）/ O 應付淨額（公式）/ P 狀態

### 操作方式
- **更新前端**：修改 index.html → 上傳到 GitHub repo 覆蓋舊檔
- **更新後端**：修改 Code.gs → 存檔 → 部署 → 管理部署作業 → ✏️ 編輯 → 版本選「新版本」→ 部署（URL 不變）
- **產出 PDF**：開 Google Sheets → 點選資料列 → 選單「🛸 勞報單」→「✅ 產出 PDF」

### 踩過的坑（重要！未來維護請注意）
1. **GCP 專案設定**：Apps Script 預設用隱藏的「預設」Cloud 專案，新建的腳本若有 Drive/Docs 等高權限 API，有登入 Google 帳號的用戶開網頁會被擋。解法是建立正式 GCP 專案（fun-labor-report）並在其中啟用 Drive API、Sheets API、Docs API，OAuth 同意畫面設為「外部＋實際運作中」。
2. **API 需手動啟用**：新 GCP 專案所有 API 預設關閉，即使 Apps Script 有寫 DriveApp 也不會自動開啟，必須在 Cloud Console 手動啟用。
3. **前端改動不需重新部署 Apps Script**，後端改動才需要。
4. **Canvas 簽名**：touch-action: none 只能加在 canvas 元素本身，不能加在外層 div，否則整個頁面無法滑動。
